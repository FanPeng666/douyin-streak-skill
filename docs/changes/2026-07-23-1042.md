# 2026-07-23-1042

## 为什么修改

1. `auto_streak.js` 重构（v2.1.0）时，`parseCookies(COOKIE_STR)` 一处调用缺失 `domain` 参数，导致 Cookie 注入失败：`browserContext.addCookies: Cookie should have a url or a domain/path pair`
2. 好友点击后聊天窗口未正确打开时，验证错误地通过 `bodyHasMsg` 兜（误判为发送成功），需增加聊天开打验证

## 修改了哪些文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `scripts/auto_streak.js` | 修复 | 第 212 行 `parseCookies(COOKIE_STR, config.COOKIE_DOMAIN)` 补传 domain |
| `scripts/auto_streak.js` | 增强 | 新增聊天打开验证：locator 点击后检查 `[contenteditable="true"]` 是否可见 |
| `scripts/auto_streak.js` | 增强 | 聊天未打开时自动坐标兜底重试 |
| `docs/changes/2026-07-23-1042.md` | 新建 | 本次变更的详细记录 |
| `CHANGELOG.md` | 更新 | 追加 v2.2.1 |

## 注意事项

- 重构导致的遗漏：函数签名变更（`parseCookies` 新增 `domain` 参数）后，旧调用未同步更新
- 之后重构应全局搜索旧调用处的所有引用后再提交
