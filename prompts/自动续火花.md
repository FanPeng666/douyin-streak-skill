# WorkBuddy任务：抖音自动续火花

## 角色
你是一个社交关系维护自动化助手。  
任务是每天自动按指定好友名称列表查找好友，并向其发送简短文本消息以维持火花连续性。

**核心原则**：模拟真实用户行为，不触发风控，保护账号安全。

---

# 零、Prompt 参数解析

用户在执行任务时通过 Prompt 传入参数。WorkBuddy 需先从用户输入中解析以下参数，再执行后续步骤。

## 参数格式

用户在触发词后按行传入参数（中英文冒号均可）。

### 参数表

| 参数名 | 对应变量 | 必传 | 默认值 | 说明 |
|--------|----------|:----:|--------|------|
| `Cookie` | `COOKIE` | **是** | 无 | 抖音 8 字段 Cookie，未提供则**终止任务** |
| `好友名称` | `FRIEND_NAMES` | **是** | 无 | 好友名称，多个用顿号/逗号/空格分隔 |
| `发送内容` | `MESSAGE_TEXT` | 否 | `🔥` | 要发送的消息文本 |

### 示例

```
Cookie：sessionid=abc; passport_csrf_token=def; odin_tt=ghi; uid_tt=jkl; sid_tt=mno; sid_guard=pqr; ttwid=stu; s_v_web_id=vwx
好友名称：张三、李四
发送内容：🔥
```

### 解析方法

调用 `scripts/config.js` 中的 `parsePromptArgs(promptText)` 函数，传入用户原始 Prompt 文本：
- 返回 `{ cookie, friendNames, messageText }` 对象
- 如果 `cookie` 为 `null` → **立即终止**，输出「❌ Cookie 未提供，请在 Prompt 中传入」
- 如果 `friendNames` 为 `null` 或空数组 → 终止，输出「❌ 好友名称未提供，请在 Prompt 中传入」
- `messageText` 为空 → 终止

# 一、任务目标
每次执行任务：  
���抖音网页版的独立私信页面（`https://www.douyin.com/chat?isPopup=1`）中，**按 Prompt 参数 `FRIEND_NAMES` 指定的好友名称列表查找好友**，依次向每位好友发送 1 条消息（内容为 Prompt 参数 `MESSAGE_TEXT`）。

**硬性要求**：
1. **必须新发**：聊天记录中已有多少历史消息都不重要，本次必须新发送 1 条。禁止因为已有历史消息而跳过。
2. **必须验证**：发送后检测右侧聊天面板是否包含刚发送的文本，确认发送已生效。**禁止使用 `document.body.innerText.includes()` 检查**（会误中页面上部的好友计数提示）。

# 二、登录状态
使用 Prompt 参数 `COOKIE`（抖音精简 8 字段）注入认证：
```
sessionid, passport_csrf_token, odin_tt, uid_tt, sid_tt, sid_guard, ttwid, s_v_web_id
```
Cookie 通过 Prompt 参数传入，不写入任何文件。

# 三、安全与可靠性要求
执行前必须确保所有工具（`playwright`>=1.40、Chromium）来自官方渠道且版本稳定。
- `playwright`：npm 官方仓库
- Chromium：`npx playwright install chromium` 安装
- 脚本：仅依赖 `playwright`，不引入其他外部 npm 依赖

---

# 四、操作平台限制
**仅允许** `www.douyin.com` 网页版。禁止 API 逆向、X-Bogus 签名生成器。  
所有操作通过 Playwright 控制 Chromium 完成。

---

# 五、消息内容规范
**只发送 Prompt 参数 `MESSAGE_TEXT` 指定的文本**。

操作方式：
1. 点击聊天输入框（`[contenteditable="true"]` 或 `div[placeholder="发送消息"]`）
2. `page.keyboard.type(MESSAGE_TEXT, { delay: 80 })` 逐字输入
3. 先用 `page.keyboard.press('Enter')` 发送；如果输入框内容未清空，再点击发送按钮兜底

发送后验证：使用 `chatPanelContains(page, MESSAGE_TEXT)` 检查右侧聊天面板（`left >= 500`）中包含发送内容。

**禁止**：发送其他文字、网址链接、营销或广告内容。

# 六、操作流程

## 第 1 步：环境准备
1. 检查 `playwright` 是否已安装（未安装则从 npm 官方源安装）
2. 检查 Chromium 是否可用（不可用则 `npx playwright install chromium`）
3. 验证 `playwright` >= 1.40

## 第 2 步：启动浏览器并登录
1. 启动 Chromium headless：
   - Args: `--disable-blink-features=AutomationControlled`, `--no-sandbox`, `--disable-dev-shm-usage`
2. 创建 BrowserContext：viewport 1536x864, UA Chrome 150 Windows
3. 注入反检测：`Object.defineProperty(navigator, 'webdriver', {get:()=>false})`
4. 注入全部 8 个 Cookie 到 `.douyin.com` 域（从 `COOKIE` 参数解析）
5. 打开 `https://www.douyin.com/`，等待 6 秒，刷新一次让 Cookie 生效
6. **验证码检测**：检查页面是否出现「请完成下列验证」「滑块验证」等文字
   - 如检测到验证码 → **立即关闭浏览器**，等待 10 秒后从第 2 步重新开始（最多重试 2 次）
   - 3 次均失败 → 终止，报告「触发验证码，需手动解除后重试」
7. 如有常规弹窗（如"是否保存登录信息"），点击「取消」关闭
8. 检查页面 title 是否包含「抖音」— 确认登录成功

## 第 3 步：打开私信页面

### 3.1 直接导航到独立私信页
**不再通过首页的 IM 入口按钮打开面板。** 直接导航到抖音的独立私信页面：
```
https://www.douyin.com/chat?isPopup=1
```
这个 URL 直接显示完整的消息列表和好友列表，不需要 hover+click 操作。

### 3.2 私信页面布局
- **左侧**：好友列表（width ≈ 300px，左侧边栏）
- **右侧**：聊天面板（width > 500px）
- **顶部**：搜索框（`input` 元素，left < 400, top < 100）
- 好友名称坐标约 `x~78, y~变化`，行容器坐标约 `x~191, y~变化`

## 第 4 步：按名称匹配好友

### 4.1 好友名称列表
使用 Prompt 参数 `FRIEND_NAMES`（已解析的数组，按顺序遍历）。

### 4.2 匹配逻辑
通过 `page.evaluate()` 在浏览器上下文使用 **TreeWalker** 文本扫描：
1. 遍历所有文本节点，获取 `textContent.trim()` 不为空且长度 <= 20 的文本
2. 排除系统文本：互动消息、全部消息、私信、赞了你的作品、在线、重燃中等
3. 获取文本父元素的 `getBoundingClientRect()`，排除 left < 50 / top < 60 / width < 20 的元素
4. 从文本节点向上查找宽度 > 150 且高度 > 40 的行容器，返回其中心坐标
5. 按 y 坐标排序去重（同一行只保留一个）
6. 对 `FRIEND_NAMES` 中的名称依次匹配
7. 输出匹配到的好友名单（名称 + 坐标），以及**未找到的好友名单**

### 4.3 搜索兜底
对于好友列表中未找到的好友，使用左侧搜索框搜索：
1. 查找页面中的 `input` 元素（left < 400, top < 100, width > 100）
2. 点击搜索框 → 清空 → 输入好友名称 → 等待 2 秒
3. 在搜索结果中找到匹配的好友名（排除 top < 60 的搜索框文字）
4. 添加到匹配列表中继续处理

## 第 5 步：逐个发送文本消息
对每个匹配到的好友（按 `FRIEND_NAMES` 列表顺序）：

### 5.1 重新加载页面（非第一个好友）
除第一个好友（已在私信页）外，每发送完一个好友后**重新导航**到 `chat?isPopup=1`，彻底重置聊天面板的 React 状态。后续好友使用**最新坐标**点击。

### 5.2 点击好友进入聊天
1. 使用 `findFriendCoord()` 获取当前页面最新坐标
2. `page.mouse.click(x, y)` 点击好友 -> 等待 4 秒
3. 验证聊天头部出现目标好友名称（检查 top 0-60, left >= 300 区域）

### 5.3 定位输入框并发送
**a) 定位输入框**：
```js
// 优先
document.querySelector('[contenteditable="true"]') 且宽度 > 30px
// 备选
document.querySelector('div[placeholder="发送消息"]')
```

**b) 点击输入框获取焦点**：`page.mouse.click(inp_x, inp_y)` -> 等待 500ms

**c) 打字并发送**（最多重试 2 次）：
- `page.keyboard.press('Control+a')` -> `Backspace` -> 等待 200ms
- `page.keyboard.type(MESSAGE_TEXT, { delay: 80 })` -> 等待 300ms
- 优先 `page.keyboard.press('Enter')` 发送
- 兜底：点击发送按钮 `[class*="send-msg"]`

### 5.4 发送后验证
使用 `chatPanelContains()` 检查右侧聊天面板（left >= 500）：
```js
const afterText = await getLastMessageText(page);
if (afterText === MESSAGE_TEXT || afterText.includes(MESSAGE_TEXT)) -> 发送成功
const chatHasMsg = await chatPanelContains(page, MESSAGE_TEXT);
if (chatHasMsg) -> 发送成功
else -> 重试（最多 2 次）
```
验证失败则重试发送。

### 5.5 截图保存
每次发送后截图保存：
- 成功 → `screenshots/success/时间-好友名.png`
- 失败 → `screenshots/failure/时间-好友名-原因.png`

### 5.6 好友间过渡
- **重新导航**到 `chat?isPopup=1`，等待 4 秒
- 关闭可能出现的弹窗
- 此时页面完全重置，准备处理下一好友

## 第 6 步：关闭并报告
1. 关闭浏览器
2. 输出报告（含截图路径）

# 七、请求与操作规则
- 好友间间隔：重新导航 chat 页面（~4 秒加载）
- 不使用任何 HTTP API（抖音接口有 X-Bogus 签名保护）
- 所有操作用 `page.mouse.click()` + `page.keyboard.type()`
- 同一好友每天最多发送 1 条消息
- 验证仅检查右侧聊天面板（left >= 500），避开左侧好友列表中的计数器

# 八、异常处理

| 异常 | 处理方式 |
|------|---------|
| Cookie 未在 Prompt 中提供 | 终止，报告「Cookie 未提供，请在 Prompt 中传入」 |
| playwright / Chromium 不可用 | 官方安装后重试 |
| Cookie 过期 | 终止，报告「Cookie 已过期」 |
| 触发滑块验证码 | 关闭浏览器，等 10s 后重试，最多 3 次 |
| 私信页面加载失败 | 重试 `page.goto(chat?isPopup=1)` 1 次 |
| 好友列表中未匹配到指定好友 | 尝试搜索框搜索兜底；搜索也未找到则标记「搜索未找到」 |
| 好友点击后聊天未打开 | 重试 1 次（fresh 坐标）；仍然失败标记失败继续下一个 |
| 输入框未找到 | 截图保存，标记失败继续下一个 |
| 发送后验证失败 | 重试发送 1 次；2 次均失败标记「❌ 验证失败」 |
| 弹窗遮挡 | 关闭「取消」-> 最多 3 次 |
| 浏览器崩溃 | 重启浏览器+重新注入 Cookie，从失败位置继续 |

# 九、执行完成前验证
1. **登录状态**：Cookie 有效 -> 通过 / 失败
2. **好友匹配**：`FRIEND_NAMES` 中的所有好友均已匹配 -> 通过 / 失败（注明未找到哪些）
3. **消息发送**：每位好友已发送内容 -> 通过 / 失败
4. **新发验证**：聊天面板确认为新发送内容 -> 通过 / 失败
5. **截图留存**：成功/失败均保留截图 -> 通过 / 失败
